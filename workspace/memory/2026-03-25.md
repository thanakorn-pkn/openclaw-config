## Mission Control — Phase 1 Brainstorming & Spec

Big planning session with Bank. Defined the full Phase 1 scope for Mission Control.

### Key Decisions
- **Concept**: Mission Control is a human-AI collaboration board, not just a generic tracker
- **Workflow**: Bank reviews SPECS (design gate), AI agents build and verify code
- **8 board statuses**: Backlog → Todo → Design → Review → In Progress → AI Review → Done / Cancelled
- **Bank's role**: Reviews specs at Design gate, NOT code. Higher leverage.
- **Review flow**: Explicit action buttons (Approve / Request Changes / Cancel)
- **AI Review**: Gemini checks code against approved spec, tests, quality
- **Gemini double-duty** (not extra columns — sub-steps within existing columns):
  - Step 1 (Design column): Gemini reads spec → posts pre-review comments in activity feed BEFORE Bank's design review
  - Step 2 (AI Review column): Gemini checks code against approved spec after coder finishes
- **Gemini CLI setup**: CLI installed and working; `gemini-2.5-pro-preview` returned 404 (model name issue) → using default Gemini model for now, to resolve later
- **`skip_design` decision (Option B chosen)**: Drop `skip_design` entirely — everything goes through the full flow. For bug fixes/small tasks, the Design step is just a one-liner spec (e.g. "Fix: search crashes on empty query"). Overhead is tiny, consistency is high. Bank still gets to review even tiny specs in 5 seconds.
- **No sprints** — pure Kanban. Roadmap view deferred.
- **Notifications via Telegram** — triggered when card hits "In Review"
- **Done/Cancelled**: visible columns but auto-filtered (7 days default)
- **Search**: global text search across all cards
- **Agent team**: Coordinator (me, Opus), Architect (Claude Code Sonnet thinking), Coder (Claude Code Sonnet thinking), Reviewer (Gemini 2.5 pro)
- **Agent identity**: flexible string IDs, Bank will define roster later
- **Per-project workflow (Option B, future)**: Each project defines which statuses it uses. Software = all 8 columns. Research = Backlog → Todo → Planning → Review → Executing → Done. Simple = Backlog → Todo → In Progress → Done. Phase 1 keeps 8 columns (software-first). Project has a `workflow` field.
